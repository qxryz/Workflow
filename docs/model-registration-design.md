# 模型注册表

模型注册和工作流节点使用是两件事。

## 设置中的模型注册

用户先在 `设置 -> 供应商` 填写 Base URL 和 API Key，并导入模型。然后在 `设置 -> 模型` 查看模型注册表，为每个模型保存一个或多个注册接口。

每个注册接口直接描述一次可执行请求：

- 任务类型，例如文生图、图生视频或语音生成；
- 请求地址、HTTP 方法和附加 Header；
- 输入材料应填入请求中的哪个参数槽位；
- 输出结果应从响应 JSON 的哪个路径读取；
- 同步、流式或异步任务；
- 异步任务 ID、查询路径、状态字段和轮询频率。

输入材料是可以拖动的卡片。把图片、视频、音频或文件卡片拖到参数槽后，节点收到同类附件时会自动填写该参数。对话接口使用 Messages 内容块；图片理解仍然属于对话请求，而图片生成使用独立特殊接口。默认请求 JSON、自定义 Header、值包装模板和数组收集方式收在高级设置中。

## 节点中的模型使用

节点只选择一个已完成登记的具体接口、填写消息并接收附件。运行时路由器读取该接口，按附件类型自动填入参数槽位。节点不会再根据模型名称或输出模态猜测路由。

注册模板可以声明少量节点常用参数。例如 Seedance 视频节点可调整时长、比例、分辨率、音频开关和 seed。这些值只覆盖当前节点，不会改写模型注册表。缺少必填附件时，调用会停止并显示缺少的材料类型。

## 兼容策略

旧配置加载时会自动生成一份可编辑的注册草稿。旧的 Endpoint、Capability 和 Schema 数据暂时保留用于迁移，不再出现在设置界面，也不再参与节点主执行路径。

每个注册接口都可以使用“重置模板”恢复供应商起始草稿。复杂供应商的异步请求头和轮询路径会尽量带入草稿，用户仍然可以按官方文档修改。
